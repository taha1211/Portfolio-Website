Since March 2018, I've worked under University of Michigan Dining Services as a Computer Consultant II (Student Programmer) creating and managing different methods and forms, data systems, and Google-Apps based projects both independentally and with designated teams. OrganizeInfo G-Suite Sheets Add-On

Using both the G-Scripts IDE and documentation, I've developed and tested an add-on that takes in user input for fields such as deposit sheet, header-keyword, and item-keyword, then scans the entire selected sheet looking for all possible events with the keyword criteria.

Each instance that is found, is labeled under another user-created sheet, with appropriate header and item information, and the user recieves a full header item list completely organized and at their disposal. This becoming highly usable with those looking to scan large data-sets quickly and organize content effectively with little to no hassle.

Data Conversion and Maintenence

A large primary day-to-day operation of my role at Michigan Dining is both data management and conversion between different mediums. This means taking large streams of data from one program or resource, and importing and configuring the data input for another resource.

Often times, this is done either through custom-built scripts that run over large data-sets, or a combination of scripting and by-hand analysis and conversion. Data sets often range in content and length from only a few hundred to thousands of possible combinations/entries.

I also work with data maintenence, by adding new user-profiles and information, and updating information throughout the system upon custom user requests and change-logs.

The primary tools used for these techniques involve using a combination of the G-Suite and Google's own native apps scripting via JavaScript, and Java implementations.
