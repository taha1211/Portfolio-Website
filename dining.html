Michigan Dining Services

Currently, I work for The University Of Michigan Dining Servies as a Computer Conusltant II (Student Developer), using primarily JavaScript, HTML, and CSS to create and manage different databases, system analytics tools, and Google Apps Scripts. Below are some more individualized approaches to the different tasks that I do on a per daily basis.

OrganizeInfo G-Suite Sheets Add-On

Using primarily the G-Suite online-IDE, I've developed a team-level tool for navigating and organizing data in order to create and run different forms of system analytics aiding database organization and efficiency. The code-base for the tool is well-commented and documented as well, allowing other developers on the team to adjust and adapt the central code to fit multiple circumstances and different data-types.

Currently, OrganizeInfo is being used to effectively help clean our system's primary operations database, which contains over 11,000 data entries. It's gone through multiple stages of testing and development and now uses a custom hash table algorithm to easily and quickly perform analytics on such a large slew of data.

Data Conversion and Maintenence

A large primary day-to-day operation of my role at Michigan Dining is both data management and conversion between different mediums. This means taking large streams of data from one program or resource, and importing and configuring the data input for another resource.

Often times, this is done either through custom-built scripts that run over large data-sets, or a combination of scripting and by-hand analysis and conversion. Data sets often range in content and length from only a few hundred to thousands of possible combinations/entries.

I also work with data maintenence, by adding new user-profiles and information, and updating information throughout the system upon custom user requests and change-logs.

All of these tools involve comprehensive work in: JavaScript, Java, HTML, CSS, VBA, Macros, Forms, and Python. They range from front end based approaches, to heavily back end oriented work depending on the circumstance.
